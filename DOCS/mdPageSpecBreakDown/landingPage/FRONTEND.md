# Simploy Frontend — File Reference

## App

| File | Purpose |
|------|---------|
| `app/page.tsx` | Root route (`/`). Imports and sequences all landing page sections in order. |
| `app/layout.tsx` | Root layout. Sets page metadata (title, description, Open Graph tags) and wraps all pages. |
| `app/globals.css` | Global styles. Defines all CSS custom properties (`--pink`, `--teal`, `--bg-page`, etc.) used throughout the app, plus smooth scroll and base body styles. |

---

## Shared UI Components (`components/ui/`)

| File | Purpose |
|------|---------|
| `Button.tsx` | Reusable button with `variant` prop — `primary` (filled pink) or `outline` (bordered pink). Accepts a `size` prop (`sm` / `md`). |
| `SectionLabel.tsx` | Small uppercase eyebrow label rendered above section headlines (e.g. "FOR EMPLOYEES"). |
| `FadeUp.tsx` | Framer Motion wrapper that animates children from `opacity: 0, y: 20` to visible on scroll. Accepts a `delay` prop for staggering. |

---

## Landing Sections (`components/landing/`)

| File | Purpose |
|------|---------|
| `Navbar.tsx` | Fixed top navigation. Adds a shadow on scroll. Contains desktop nav links (anchored to page sections), Sign In / Get Started buttons, and a mobile hamburger drawer. |
| `Hero.tsx` | Top hero section. Includes the eyebrow badge, headline, subheadline, dual CTAs (employee / employer), and a fully div-based employer dashboard mockup with stat cards, job table, and insights bars. |
| `SocialProof.tsx` | Thin strip of four social proof metrics (skills mapped, match accuracy, companies, talent value). |
| `HowItWorks.tsx` | Three-card section explaining the Career Marketplace, Gap Simulator, and Action Engine layers. Cards animate in with a stagger on scroll. Includes a flow connector strip at the bottom. |
| `ForEmployees.tsx` | Two-column section targeting employees. Left: feature list (AI matching, skill gaps, internal mobility, talent graph). Right: div-based employee dashboard mockup with job match cards and a skill gap panel. |
| `ForEmployers.tsx` | Two-column section targeting employers. Left: div-based Gap Simulator mockup with what-if tabs, stat cards, supply/demand bars, and risk table. Right: feature list. |
| `Pricing.tsx` | Three-tier pricing section (Individual / Starter / Enterprise). Starter card is featured with a pink border and "Most Popular" badge. Cards animate in on scroll. |
| `About.tsx` | Company mission section with two paragraphs and three team member avatar cards. |
| `Footer.tsx` | Dark footer with four columns — brand tagline, Product links, Company links, Legal links — and a bottom copyright bar. |

## Config

| File | Purpose |
|------|---------|
| `tailwind.config.ts` | Extends Tailwind with brand colors (`brand-pink`, `brand-teal`, `brand-dark`, etc.) and sets Inter as the default sans font. |

---

## Do NOT push to GitHub

| Path | Reason |
|------|--------|
| `node_modules/` | Auto-generated by `npm install`. Already in `.gitignore` by default — never commit this. |
| `.next/` | Next.js build output. Already in `.gitignore` — no need to push, it gets regenerated on deploy. |
| `.env` / `.env.local` | If you add any API keys or secrets later, keep these out of git. Add them to `.gitignore` and use your hosting platform's environment variable settings instead. |

Everything else in `frontend/` is safe to push.
